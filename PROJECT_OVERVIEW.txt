프로젝트명: 모바일 협업 노트 앱

1. 프로젝트 개요
간단하고 직관적인 모바일 전용 협업 노트 앱으로, 사용자가 언제 어디서든 팀원들과 함께 아이디어를 기록하고 관리하며, 실시간으로 협업할 수 있도록 돕습니다. 사용자 계층별로 차등화된 협업 기능을 제공하며, **실시간 협업**을 핵심 차별점으로 내세웁니다.

2. 개발 목표 및 기간
- 단기 목표 (MVP): 2.5개월 (약 10주) 내에 핵심 필기 기능, 회원 인증 및 클라우드 동기화, 그리고 **제한된 실시간 협업 기능**의 MVP 버전을 구현하여 출시 가능한 상태를 만듭니다.
- 장기 목표: **무제한 프로젝트/협업 기능, 고급 협업 기능 (변경 내용 추적, 편집 권한 관리)**, 인앱 결제 시스템 등을 순차적으로 개발하여 비즈니스 모델을 완성합니다.

3. 주요 기능 및 사용자 계층별 차등
앱은 비회원, 회원(무료 가입), 프로(Pro) 구독자 세 가지 사용자 계층을 지원합니다.

3.1. 비회원 (Non-member)
- 기본 필기 기능 (노트 생성/수정/삭제, 목록 보기)
- 기본 서식 (볼드, 이탤릭, 밑줄 등)
- 로컬 저장 공간 무제한
- 협업 불가능, 프로젝트 개념 없음 (개인 공간)

3.2. 회원 (무료 가입)
- 모든 비회원 기능 포함
- 클라우드 동기화 (Supabase 연동)
- 클라우드 저장 공간 제한 (예: 1GB 또는 일정 개수의 노트)
- 분할 화면 기능
- **제한된 실시간 협업 기능 (최대 2명 동시 편집, 댓글 기능, 읽기 전용 공유)**
- 프로젝트 기능 (제한: 최대 3개 프로젝트, 프로젝트당 최대 5명 멤버 초대)
- 버전 기록 (제한: 최근 24시간 동안의 노트 변경 기록 확인)

3.3. 프로(Pro) 구독자 (유료)
- 모든 회원 기능 포함
- 클라우드 저장 공간 무제한
- **고급 협업 기능 (무제한 동시 편집, 편집 권한 관리, 변경 내용 추적)**
- 프로젝트 기능 (무제한 프로젝트, 무제한 멤버 초대)
- 버전 기록 (무제한: 전체 변경 이력 확인 및 복원)
- 추가 Pro 기능 (프로젝트별 진행 상황 대시보드, 프로젝트 템플릿 생성 및 공유)

4. 기술 스택
- 개발 프레임워크: React Native (JavaScript/TypeScript)
- 상태 관리: Redux Toolkit 또는 Zustand
- 로컬 데이터베이스: WatermelonDB 또는 react-native-sqlite-storage
- 클라우드 백엔드/BaaS: Supabase (PostgreSQL, Auth, Edge Functions, Realtime)
- 기타 주요 라이브러리: react-navigation, react-native-reanimated, react-native-gesture-handler, react-native-iap 등

5. 아키텍처 주요 특징
- 하이브리드 데이터 전략: 로컬 DB와 Supabase 클라우드 DB를 함께 사용하여 오프라인 지원 및 클라우드 동기화 구현.
- 사용자 계층별 기능 게이팅: 사용자 로그인 및 구독 상태에 따라 기능 접근을 제어.
- 프로젝트 개념 도입: 노트를 체계적으로 관리하고 협업을 위한 상위 단위 제공.
- **실시간 협업 기능 통합**: Supabase Realtime 기능을 활용한 실시간 동기화 및 충돌 처리.
- 인앱 결제 연동: Apple/Google 인앱 결제 시스템을 통한 구독 관리.

6. 개발 프로세스 및 협업
- 개발 기간: 약 10주 (2.5개월)
- 개발 인원: 2인
- 개발 방법론: 스프린트 기반의 애자일 개발
- 역할 분담: 프론트엔드/UI 중심 개발자 A, 백엔드/데이터/통합 중심 개발자 B (유동적)
- Git 워크플로우: `main`, `develop`, `feature/{기능이름}` 브랜치 전략, Pull Request 기반 코드 리뷰.
- AI 개발 도구 활용: Cursor, Gemini 등을 적극 활용하여 개발 생산성 향상.
- 의사소통: 정기적인 스크럼, API 및 데이터 구조 합의, 코드 리뷰를 통한 긴밀한 협업.
